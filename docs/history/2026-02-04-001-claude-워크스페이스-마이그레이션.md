# 워크스페이스 마이그레이션

## Why

jeongph-workspace가 Claude Code 최상위 지침과 워크스페이스 루트 역할을 동시에 담당하고 있었으나, lighthouse 레포와 역할이 분산되어 관리가 비효율적이었다. 하나의 워크스페이스 루트로 통합할 필요가 있었다.

## How

1. jeongph-workspace 전체를 tar.gz로 백업
2. lighthouse/guide/ 3개 파일(Claude_Code.md, tmux.md, gh.md)을 handy/guide/로 이관
3. lighthouse에서 불필요 디렉토리(guide/, development/) 제거
4. `git merge --allow-unrelated-histories`로 jeongph-workspace 커밋 히스토리를 lighthouse에 통합
5. .gitignore 재작성 — 워크스페이스 루트 + conventions/, docs/history/ 허용 구조
6. git-commit-convention.md 수정 — other, init 타입 제거, chore 타입 추가
7. CLAUDE.md 전면 재작성 — lighthouse 맥락에 맞게 제목, 개요, Git 정책, 레포 목록 등
8. README.md 수정 — 워크스페이스 구조 섹션 추가
9. lighthouse를 /home/jeonguk/lighthouse로 이동, handy와 helpful-feeds를 그 안으로 배치
10. lighthouse, handy 원격 push

## What

- jeongph-workspace의 커밋 히스토리를 lighthouse에 통합
- lighthouse를 단일 워크스페이스 루트(`/home/jeonguk/lighthouse/`)로 전환
- CLAUDE.md, .gitignore, README.md, git-commit-convention.md 재작성
- jeongph-workspace 백업: `/home/jeonguk/jeongph-workspace-backup-20260204.tar.gz`
- jeongph-workspace GitHub 레포 archive 및 로컬 삭제는 별도 확인 후 진행
