# 워크스페이스 디렉토리 구분 개선

## Why
워크스페이스 루트에 메타 파일(CLAUDE.md, conventions/ 등)과 하위 프로젝트(handy/, helpful-feeds/)가 동일 레벨에 혼재되어 있어 구조적 구분이 불명확했다. 프로젝트가 늘어날수록 루트가 복잡해지므로, 메타와 프로젝트를 명확히 분리할 필요가 있었다.

## How
1. `projects/` 디렉토리를 생성하고 `handy/`, `helpful-feeds/`를 이동
2. `CLAUDE.md`의 레포지토리 목록 경로에 `projects/` 접두사 추가 후 즉시 커밋 & 푸시
3. `README.md`의 워크스페이스 구조도를 새 구조로 갱신 후 커밋 & 푸시
4. 각 프로젝트의 git remote, 워크스페이스 git status 정상 확인

## What
- `projects/` 디렉토리 신규 생성
- `handy/` → `projects/handy/`, `helpful-feeds/` → `projects/helpful-feeds/` 이동
- `CLAUDE.md` 레포지토리 목록 경로 변경 (`projects/` 접두사)
- `README.md` 워크스페이스 구조도 갱신
