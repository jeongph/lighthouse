# 히스토리 문서 컨벤션 수립

## Why

히스토리 문서에 정적 문서 생성기를 붙이면 가독성이 좋아지는데, 기존 히스토리 문서에는 YAML frontmatter가 없어 정적 사이트 생성기와 호환되지 않았다. 또한 히스토리 작성 규칙이 CLAUDE.md에 인라인으로만 존재해 상세 템플릿이 부족했다.

## How

1. `conventions/history-convention.md` 신규 작성 — frontmatter 스펙, 본문 구조, 전체 템플릿 포함
2. CLAUDE.md의 히스토리 섹션을 간략화하고 컨벤션 문서 참조로 변경 (커밋 가이드라인과 동일한 패턴)
3. 혼동을 주는 002 히스토리 파일(프로젝트 디렉토리 구분) 삭제
4. 기존 001 히스토리 파일을 새 컨벤션 형식으로 변환

## What

- `conventions/history-convention.md` 신규 생성 — frontmatter 필드(title, description, date, author, project, tags), 문서 제목(h1), Why/How/What 본문 구조 정의
- `CLAUDE.md` 히스토리 섹션 간략화 및 컨벤션 참조 연결
- `docs/history/2026-02-04-002-claude-프로젝트-디렉토리-구분.md` 삭제
- `docs/history/2026-02-04-001-claude-워크스페이스-마이그레이션.md` 새 컨벤션 형식으로 변환
